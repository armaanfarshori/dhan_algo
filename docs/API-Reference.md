# API Reference

All endpoints are served by the aiohttp web server on `http://localhost:8765` (configurable via `WEBHOOK_PORT`). All responses are `application/json`. CORS headers are set to `*` â€” no browser restrictions apply.

No authentication is required. If you expose the server beyond localhost, protect it with a reverse proxy.

---

## Endpoints at a Glance

| Method | Path | Description |
|---|---|---|
| `GET` | `/api/status` | Strategy state, uptime, mode, warmup progress |
| `GET` | `/api/risk` | P&L snapshot, violations, halt state |
| `GET` | `/api/signals` | Last 50 signals, newest first |
| `GET` | `/api/funds` | Available funds from DhanHQ |
| `GET` | `/api/positions` | Open positions from DhanHQ |
| `GET` | `/api/scalper` | Scalper-specific state (RSI, OCO, ATM) |
| `GET` | `/api/instruments` | NIFTY expiry list from instrument master |
| `POST` | `/postback` | DhanHQ order postback webhook receiver |

Additionally:
- `GET /` â€” serves the React dashboard (`dashboard/dist/index.html`) or fallback (`static/index.html`)
- `GET /health` â€” liveness probe

## GET /api/status

Returns current strategy state, uptime, trading mode, and (for the SMA strategy) warmup progress.

**Request:** No parameters.

**Response â€” scalper mode:**

```json
{
  "mode": "PAPER",
  "client_id": "1234567890",
  "uptime_seconds": 3742,
  "strategy_name": "Options_Scalper",
  "strategy_running": true,
  "orders_placed": 2,
  "position": 75,
  "entry_price": 47.5,
  "warmup": {
    "ready": true
  }
}
```

**Response â€” SMA mode (during warmup):**

```json
{
  "mode": "PAPER",
  "client_id": "1234567890",
  "uptime_seconds": 120,
  "strategy_name": "SMA_9_21_Reliance",
  "strategy_running": true,
  "orders_placed": 0,
  "position": 0,
  "entry_price": 0.0,
  "warmup": {
    "fast_current": 9,
    "fast_required": 9,
    "slow_current": 14,
    "slow_required": 21,
    "ready": false
  }
}
```

**Fields:**

| Field | Type | Description |
|---|---|---|
| `mode` | string | `"PAPER"` or `"LIVE"` |
| `client_id` | string | DhanHQ client ID in use |
| `uptime_seconds` | int | Seconds since process started |
| `strategy_name` | string | Config `name` field of the active strategy |
| `strategy_running` | bool | Whether the strategy loop is active |
| `orders_placed` | int | Orders placed this session (resets on restart) |
| `position` | int | Net open quantity (0 = flat) |
| `entry_price` | float | Entry price of the current position (0 if flat) |
| `warmup.ready` | bool | Whether the strategy has enough data to trade |

## GET /api/risk

Returns the real-time risk snapshot. Updated every 30 seconds by the background risk monitor.

**Request:** No parameters.

**Response â€” normal state:**

```json
{
  "realised_pnl": 320.50,
  "unrealised_pnl": -45.00,
  "total_pnl": 275.50,
  "open_positions": 1,
  "halted": false,
  "halt_reason": "",
  "violations": [],
  "last_checked": "2026-05-06T10:32:15.421083"
}
```

**Response â€” halted state:**

```json
{
  "realised_pnl": -4200.00,
  "unrealised_pnl": -900.00,
  "total_pnl": -5100.00,
  "open_positions": 1,
  "halted": true,
  "halt_reason": "Daily loss â‚ą5,100 exceeds limit â‚ą5,000",
  "violations": [
    "Daily loss â‚ą5,100 exceeds limit â‚ą5,000"
  ],
  "last_checked": "2026-05-06T13:12:45.001234"
}
```

**Fields:**

| Field | Type | Description |
|---|---|---|
| `realised_pnl` | float | Realised P&L in INR from DhanHQ positions API |
| `unrealised_pnl` | float | Unrealised P&L in INR from DhanHQ positions API |
| `total_pnl` | float | `realised_pnl + unrealised_pnl` |
| `open_positions` | int | Count of positions with non-zero net quantity |
| `halted` | bool | Whether all new orders are blocked |
| `halt_reason` | string | Human-readable reason for the halt |
| `violations` | array | List of active rule violations |
| `last_checked` | string | ISO 8601 timestamp of last risk evaluation |

## GET /api/signals

Returns the last 50 signals generated by the active strategy, sorted newest-first.

**Request:** No parameters.

**Response:**

```json
[
  {
    "action": "BUY",
    "price": 47.50,
    "reason": "CALL | RSI 28.4 | BEP â‚ą48.23 | T â‚ą53.23 S â‚ą42.50",
    "timestamp": "2026-05-06T10:35:02.114523"
  },
  {
    "action": "EXIT",
    "price": 0,
    "reason": "OCO TRADED",
    "timestamp": "2026-05-06T10:52:18.008741"
  }
]
```

**Fields:**

| Field | Type | Description |
|---|---|---|
| `action` | string | `"BUY"`, `"SELL"`, `"EXIT"`, or `"HOLD"` |
| `price` | float | Price at signal time. `0` for automated exits where the price is determined by the OCO fill. |
| `reason` | string | Human-readable description of the signal trigger. For scalper entries includes direction, RSI value, breakeven premium, and OCO levels. |
| `timestamp` | string | ISO 8601 UTC timestamp of signal generation. |

## GET /api/funds

Proxies the DhanHQ `/fundlimit` endpoint. Returns the live available balance. Requires valid API credentials â€” returns an error if the API call fails.

**Request:** No parameters.

**Response â€” success:**

```json
{
  "ok": true,
  "data": {
    "dhanClientId": "1234567890",
    "availabelBalance": 48250.00,
    "sodLimit": 50000.00,
    "collateralAmount": 0.0,
    "receiveableAmount": 0.0,
    "utilizedAmount": 1750.00,
    "blockedPayoutAmount": 0.0,
    "withdrawableBalance": 48250.00
  }
}
```

**Response â€” API error:**

```json
{
  "ok": false,
  "error": "[DHAN-1002] Invalid access token"
}
```

HTTP status 503 is returned on error.

## GET /api/scalper

Returns the internal state of the Options Scalper strategy. Returns a 404 error object if the active strategy is not the scalper.

**Request:** No parameters.

**Response â€” scalper active, in position:**

```json
{
  "state": "IN_POSITION",
  "oco_state": "IN_POSITION",
  "entry_premium": 47.50,
  "breakeven_premium": 48.23,
  "last_rsi": 28.4,
  "oco_order_id": "112345678901234",
  "option_sid": "98765",
  "active_expiry": "2026-05-08",
  "current_atm": {
    "strike": 24500.0,
    "expiry": "2026-05-08",
    "CE_sid": "98765",
    "PE_sid": "98766"
  },
  "orders_placed": 1,
  "master_loaded": true
}
```

**Response â€” scalper active, flat:**

```json
{
  "state": "FLAT",
  "oco_state": "FLAT",
  "entry_premium": 0.0,
  "breakeven_premium": 0.0,
  "last_rsi": 52.1,
  "oco_order_id": null,
  "option_sid": null,
  "active_expiry": "2026-05-08",
  "current_atm": null,
  "orders_placed": 0,
  "master_loaded": true
}
```

**Response â€” wrong strategy:**

```json
{
  "ok": false,
  "error": "Not a scalper strategy"
}
```

HTTP status 404 is returned when the SMA strategy is active.

**Fields:**

| Field | Type | Description |
|---|---|---|
| `state` | string | Strategy state machine: `"FLAT"`, `"ENTERING"`, `"IN_POSITION"` |
| `oco_state` | string | Same values â€” mirrors `state` but persists for dashboard display |
| `entry_premium` | float | Option premium at entry (0 if flat) |
| `breakeven_premium` | float | Minimum exit premium to cover all charges (0 if flat) |
| `last_rsi` | float | Most recently computed RSI-14 value on NIFTY index |
| `oco_order_id` | string or null | DhanHQ order ID of the active OCO order |
| `option_sid` | string or null | DhanHQ security ID of the option held |
| `active_expiry` | string | Currently targeted expiry date (`YYYY-MM-DD`) |
| `current_atm` | object or null | ATM strike and CE/PE security IDs resolved at last entry |
| `orders_placed` | int | Entries made this session |
| `master_loaded` | bool | Whether the instrument master has finished loading |

## GET /api/instruments

Returns the NIFTY expiry lists from the in-memory instrument master. Returns a `loaded: false` response if the master has not yet finished loading (typically within the first 5 seconds of startup).

**Request:** No parameters.

**Response â€” master loaded:**

```json
{
  "loaded": true,
  "expiries": [
    "2026-05-08",
    "2026-05-15",
    "2026-05-22",
    "2026-05-29",
    "2026-06-26",
    "2026-07-30"
  ],
  "weekly": [
    "2026-05-08",
    "2026-05-15",
    "2026-05-22",
    "2026-05-29"
  ],
  "monthly": [
    "2026-06-26",
    "2026-07-30"
  ],
  "nearest": "2026-05-08",
  "active": "2026-05-08"
}
```

**Response â€” master not yet loaded:**

```json
{
  "loaded": false,
  "expiries": [],
  "nearest": null
}
```

Only available when the Options Scalper is the active strategy. Returns 404 for SMA strategy.

**Fields:**

| Field | Type | Description |
|---|---|---|
| `loaded` | bool | Whether the instrument master has finished loading |
| `expiries` | array | All upcoming NIFTY option expiry dates, sorted ascending |
| `weekly` | array | Expiries flagged as weekly (`SEM_EXPIRY_FLAG = "W"`) |
| `monthly` | array | Expiries flagged as monthly (`SEM_EXPIRY_FLAG = "M"`) |
| `nearest` | string or null | First expiry on or after today |
| `active` | string | The expiry currently being traded |

## POST /postback

Receives DhanHQ order update postback events. Configure this URL in the DhanHQ developer portal as your webhook endpoint.

**Request body (sent by DhanHQ):**

```json
{
  "dhanClientId": "1234567890",
  "orderId": "112345678901234",
  "correlationId": "",
  "orderStatus": "TRADED",
  "transactionType": "SELL",
  "exchangeSegment": "NSE_FNO",
  "productType": "MARGIN",
  "orderType": "LIMIT",
  "validity": "DAY",
  "tradingSymbol": "NIFTY-24500CE-08May2026",
  "securityId": "98765",
  "quantity": 75,
  "disclosedQuantity": 0,
  "price": 53.23,
  "triggerPrice": 53.23,
  "afterMarketOrder": false,
  "filled": 75,
  "pending": 0,
  "averagePrice": 53.23
}
```

**Response â€” success:**

```json
{
  "ack": "ok"
}
```

**Response â€” malformed body:**

```json
{
  "error": "Expecting value: line 1 column 1 (char 0)"
}
```

HTTP status 400 is returned on parse errors.

The postback handler currently logs the event. Future versions will use postback events to update position state and trigger OCO placement without polling.

## GET /health

Simple liveness probe. Returns immediately without touching any strategy state.

**Request:** No parameters.

**Response:**

```json
{
  "status": "ok",
  "paper": true
}
```

Use this endpoint for uptime monitoring and health checks in systemd or container orchestration.
